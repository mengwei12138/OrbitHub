# 页面约定

> 本文件定义了前端常见交互类型的 Schema 模版和案例。新建页面时，可按需复制对应交互类型的模版到页面 swimlane.yaml 中。

---

## 页面代码生成规范

### 一、生成前必须读取的文件

**生成页面或页面组件代码前，只允许读取该页面 AGENTS.md 中索引的以下文件，不允许读取任何其他来源。**

| 文件          | 作用               |
| ------------- | ------------------ |
| swimlane.yaml | 页面交互逻辑定义   |
| Figma 链接    | 页面样式和布局设计 |
| OpenAPI 文件  | 接口定义和数据模型 |
| 需求文件      | 业务规则和功能描述 |
| 验收文件      | 验收标准和测试依据 |

> **约束**：如果某个索引文件内容为空、暂无或无法访问，标记为"跳过"，不得自行从其他来源寻找替代。

### 二、生成依据优先级

#### 样式还原

**声明**：只允许使用该页面 AGENTS.md 索引的 Figma 设计稿作为首要依据。Figma 中未定义的样式，才允许从该页面 AGENTS.md 索引的其他文件依次补充。

| 优先级 | 依据          |
| ------ | ------------- |
| P1     | Figma 链接    |
| P2     | 验收文件      |
| P3     | 需求文件      |
| P4     | swimlane.yaml |
| P5     | OpenAPI 文件  |

#### 交互还原

**声明**：只允许使用该页面 AGENTS.md 索引的泳道图作为首要依据。泳道图中未定义的交互，才允许从该页面 AGENTS.md 索引的其他文件依次补充。

| 优先级 | 依据          |
| ------ | ------------- |
| P1     | swimlane.yaml |
| P2     | 验收文件      |
| P3     | 需求文件      |
| P4     | Figma 链接    |
| P5     | OpenAPI 文件  |

#### 接口逻辑

**声明**：只允许使用该页面 AGENTS.md 索引的 OpenAPI 文件作为首要依据。OpenAPI 中未定义的接口，才允许从该页面 AGENTS.md 索引的其他文件依次补充。

| 优先级 | 依据          |
| ------ | ------------- |
| P1     | OpenAPI 文件  |
| P2     | 验收文件      |
| P3     | swimlane.yaml |
| P4     | 需求文件      |
| P5     | Figma 链接    |

#### 组件复用

**声明**：优先使用项目已有组件。依次按页面私有组件、项目公共组件、项目规范、React 最佳实践 的顺序复用。

| 优先级 | 依据                                 |
| ------ | ------------------------------------ |
| P1     | 页面 AGENTS.md 所在目录的 components |
| P2     | 项目公共 components                  |
| P3     | 项目组件规范                         |
| P4     | React 最佳实践                       |

## 文件生成逻辑

### 需求类文件生成顺序

1. **需求文件** → 生成 → **requirements.feature**（需求验收文件）
2. **requirements.feature** → 生成 → **requirements.steps.tsx**（需求 BDD Step 骨架）

### 测试类文件生成顺序

1. **测试用例文件** → 生成 → **test.feature**（测试用例验收文件）
2. **test.feature** → 生成 → **test.steps.tsx**（测试 BDD Step 骨架）

### 交互类文件生成顺序

1. 读取 **需求文件** → 获取业务场景和交互流程
2. 读取 **OpenAPI 文件** → 获取 API 接口定义（method、url、params、request body）
3. 两者结合 → 生成 **swimlane.yaml**（交互逻辑定义）
4. **swimlane.yaml** → 生成 → **swimlane.svg**（交互泳道图）

### 说明

- **占位内容**：指自动生成的骨架代码，包含 Given-When-Then 结构但无具体实现，仅供后续填充
- **生成顺序**：只有当前置文件存在且非空时，才生成后续文件
- **跳过规则**：若前置文件内容为空、暂无或无法访问，标记为"跳过"，不生成后续文件
